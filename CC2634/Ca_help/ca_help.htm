Customer Assistance to Concept Error Messages

The following Error Ids are generated by the Online Interface, OLI. They may have different sub-errors which will be displayed in the message window associated with it.

Error Id:     OLI-21707

Description:    EFB Mgr service on PLC failed! See Message window for details

 -1250    E_EFB_FP_TYPE_WRONG

Possible causes:

EMUQ loadable is missing from the configuration and the logic in the project contains floating point operation.

Solution:

Install EMUQ loadable. This loadable is only needed with PLC type 140CPUx130xS.

 

Error Id:         OLI-21705

Description:    FBD CG Service on PLC failed! See Message window for details. One of the following will be displayed in the message window: #851,#857,#877,#879

-851     Invalid operation code

Possible cause:

Possibly your PLC/Exec does not support Interrupt sections or 'Secure Applications'

Solution:

Interrupt sections or 'Secure Applications' require a CPU 434-12A or CPU 534-14A with the latest Exec.

-857    Data length mismatch

Possible causes: Concept versions 2.2 SR1

SFC sections contain steps with Qualifiers and located or unlocated time variables.

Solution:

Upgrade to SR2 or higher.

Text compiler was unable to compile a complex logic.

Solution:

Simplify the logic.  

-877     PLC Program data area too small

Possible cause:

Configured Data Area in Memory Statistics is too small

Solution:

Increase Configured Data Area

 -879     PLC Program data area too big

Possible cause:

Datatypes exceeds 64K

Solution:

Restructure large Datatypes

Error Id:        OLI-21702

Description:    General PLC service failed! See Message window for details.

Detail:    This error message is generated by the PLC. Also other errors might result from this subsystem

-201    Download changes: PLC out of memory

Possible causes:

One or more sections selected for a 'download changes' exceeds the maximum data size.

The PLC memory is too fragmented due to too many 'download changes'.

Too many variables exist or the size of the variables exceeds the maximum data size.

Too many initial values were recently introduced.

The project is too big.

Solution:

Try the 'memory prediction' to identify a section which is recommended for optimization and choose 'Optimize Section' available in the project browser. Sections optimized in this way can be transferred to the PLC via 'Download Changes'.

Identify unused variables in the variable editor, delete them and choose 'Optimize Project'. This requires a complete 'Download'.

Reduce the number of initial values.

Adjust (increase/decrease dependent on the circumstances) the amount of available memory for 'Global Data' (Variables) in the 'PLC Selection' dialog.

If the project is too big or the PLC memory seems to be fragmented try a complete 'Download'.

Try an export/import via ASCII-file using the 'Concept Converter' following with a complete 'Download'. ASCII conversion leaves a memory-optimized project.

Identify memory-consuming DFB inputs/outputs and consider to use inouts.

Use a bigger PLC.

Error Id:        OLN-20801

Description:    Error generating code download for the IEC program

Detail:    This error message is generated by the Online subsystem. Also other errors might result from this subsystem

-806    Call to object database failed

Possible causes:

DFBs used in the project have not been saved in the proper directory.

Path for your project is too long

Minor database corruption

Solution:

In the case of minor database corruption close Concept and delete q1 and q2 files of the project. If database is still corrupted then export and re-import the project using the Concept Converter.

Error Id:     OL9-11543

Description: Invalid NETBIOS command (ModbusPlus only)

Detail: This error message will be generated when the communication between the programming unit and the PLC couldn't be established or has been interrupted.

Possible causes:

Incorrect destination addressing/routing

Break in communication cable or disconnected cable

MB+ Device not present or MB+ Device Driver not present or not installed

Solutions:

Check the communication addressing setup and network for above possibilities.

Error Id:         STOP-CODE

Description:    PLC has stopped for one reason. One of the following will be displayed in the status window: #0080,#4000

0080     Watchdog timer has expired

Possible causes:

Time consuming loop or endless loop in a ST/IL section.

Scan time of program is too long for the current watchdog time.

Solution:

For debug purposes enable temporarily the loop control in the 'Code Generator Options' Online events will help you to find the location of the excessive loop.

Increase the watchdog time using the configurator dialog.

4000     Bad I/O traffic cop table

Details:

Appears when attempting to start the PLC after downloading. It indicates an invalid or faulty I/O Map.

Possible causes:

'Disable all writes from NOEs/NOMs' checked in 'Quantum Security Parameters' and DIO configured through a NOM module

'Enable Write Restriction' box checked in 'Quantum Security Parameters' and DIO configured through the native Modbus Plus port BUT the DIO drop address is not in the write restriction table

NOG module with configured I/O with 'Disable all writes from NOEs/NOMs' checked in 'Quantum Security Parameters'

Solution:

Move DIO to native Modbus Plus port and add DIO drop Modbus Plus address to Write Restriction Table (if 'Enable Write Restriction' box is checked).

Uncheck the 'Disable all writes from NOEs/NOMs' if write blocking is not necessary.
